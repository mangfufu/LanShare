# Changelog

## v4.0.0 (2026-07-24)

### 独立论坛（2026-07-24）

- 增加 `/forum` 独立论坛页面和 `/forum/post/<编号>` 帖子详情页；主页面左侧导航与小论坛标题栏均可进入
- 主页面小论坛改为简版预览，只显示按最新回复排列的 5 条标题摘要、分类、作者、回复数与主要反应
- 帖子列表增加热度、最新回复、最新发布三种排序，以及分类、关键词、我的帖子、我参与的、我的收藏和未读讨论筛选
- 增加匿名发帖；匿名帖在普通列表和详情中隐藏账号身份，匿名发帖者的后续楼层显示为“匿名楼主”
- 增加投票帖子：支持 2–8 个单选项、不限时 / 1 / 3 / 7 天及自定义 1 小时至 365 天截止；游客也可投票，每个账号一票且首次投票后最多改投一次
- 帖子列表与主页面小论坛增加投票标识，详情页显示各选项票数、比例、当前选择与准确截止状态
- 左侧“逛逛分区”增加“投票”筛选标签，可集中查看全部分类中的投票帖子
- 论坛发帖对话框中的分类、投票时长和自定义单位下拉框统一为墨韵 UI，自绘箭头并预留文字间距
- 完整论坛支持帖子编辑与删除、管理员置顶与锁定、楼层编辑与软删除、定向回复及帖子/楼层 Emoji 反应
- 未读状态改为账号级已读位置，打开帖子后清除；浏览量按账号对同一帖子去重
- 论坛实时刷新改用服务端事件流，主页面保留 30 秒低频轮询作为断线兜底
- `forum.db` 自动迁移到结构版本 5，包含分类、收藏、匿名、置顶、锁定、浏览、回复统计、已读位置、楼层反应、投票选项、账号投票记录及改投次数；已有帖子与楼层保持不变
- 独立论坛按主页面墨韵 UI 重构：复用同一日间/夜间色板、纸纹、左侧竖排导航、直角细线、透明按钮和无外框反应控件，并保留宽屏与窄屏布局

### 账号与权限（2026-07-24）

- 增加独立登录/注册页，注册字段为账号、密码和姓名；账号限制 3–10 字符，密码限制 6–16 字符
- 新账号默认游客；系统尚无管理员时，从 `127.0.0.1` 或 `::1` 注册的首个账号成为初始管理员
- 初始管理员增加不可变保护标记，角色和启用状态不可被其他管理员修改；现有数据库自动保护最早创建的管理员
- 管理员可重置其他非初始管理员账号的密码；重置后目标账号的全部登录会话立即失效，初始管理员密码不允许由其他管理员重置
- 管理员可修改其他账号的真实姓名，并同步更新账号关联的论坛作者、当前聊天和文件创建者显示；本人姓名仍从账户中心修改
- 宽屏最右侧增加 68px 在线状态条，固定在五个悬浮按钮上方，以单行“头像、账号 ID、绿点”显示；详细身份与设备数移入悬停提示，停止访问约 45 秒后离线
- 管理员可在账号管理中删除非初始管理员账号，删除前二次确认并清除该账号的全部登录会话
- 注册页增加可选邀请码；管理员可设置一个长期可重复使用的邀请码，使用有效邀请码注册时直接获得可编辑角色，更换或停用后旧码立即失效
- 增加游客、可编辑、管理员三级服务端权限，管理员可在账户中心调整角色并立即作用于现有会话
- 增加 12 小时普通会话与 30 天“记住我”会话，Cookie 使用 HttpOnly、SameSite=Strict；两种会话均可在服务或浏览器重启后从 SQLite 恢复
- 8080 正式版与 8082 测试版改用不同的登录 Cookie 名称，避免同一浏览器切换端口时互相覆盖登录状态
- 密码使用 scrypt 随机盐哈希保存，登录失败增加按 IP 与账号组合的限速
- 游客可通过独立站内预览流查看图片、音频和视频，但不能访问下载接口、通用原文件接口、上传或修改文件
- 论坛帖子、楼层回复、表情反应及聊天身份改为登录账号；旧浏览器昵称不再作为服务端授权依据
- 账号数据保存在 `auth.db`，不进入代码同步或自动备份；自定义安全预览与水印留待后续实现

### 新增

- 增加账号级帖子收藏；列表和详情均可收藏或取消，并增加“我的收藏”筛选，游客账号也可使用
- 增加“我参与的”筛选，同时包含本人发布和本人回复过的帖子
- 新帖与回复增加浏览器本地草稿保留，按登录账号和帖子隔离；成功发布后自动清除
- 增加 DOCX 站内预览：服务端转换正文、列表、表格和内嵌图片，在现有预览弹窗的隔离 iframe 中显示
- DOCX 转换内容执行 HTML 白名单清理、无脚本 CSP、源文件/输出体积限制及有总量上限的内存缓存；游客可预览转换页面但不能读取原 DOCX 文件流
- 文件列表和宫格增加 Shift 连续范围多选，范围遵循当前排序、搜索和状态筛选结果，复选框同样支持
- 工作区单选图片、视频或音频时按需显示媒体详情：分辨率、像素、帧率、码率、时长、封装、编码、采样率、声道、色彩与位深等
- 媒体详情使用 Sharp / ffprobe 按需探测，并以文件路径、大小和修改时间建立有上限的内存缓存，避免目录加载时批量扫描
- 宽屏右侧增加工作区，实时展示进行中的项目、选中文件详情、创建者与当日上传/下载/删除文件数
- 在线状态条不再占用第三列宽度，工作区和小论坛恢复原宽度；人数超出可用高度时仅在状态条内部滚动
- 增加“小论坛”：初版支持发帖、帖子内楼层回复、每次加载五楼、Emoji 反应及实时刷新；现已改为独立论坛的简版入口
- 楼层支持定向回复并显示被回复的楼层与作者，点击引用可定位原楼层
- 帖子删除权限按登录账号判断：发帖者删除自己的帖子，管理员删除全部帖子
- 帖子回复红点改为当前浏览器的未读提醒，展开后清除，后续出现新回复时重新提示

### 优化

- 独立论坛恢复“左侧筛选与分类 + 主帖子区”两栏结构，右侧“讨论概况 / 分类说明”保持移除；窄屏时左栏折为横向可滑动入口
- 分类沿用稳定标识并更新为“茶水间 / 一起开工 / 敲黑板 / 求个支援”，旧数据库启动后自动更新显示名，不改变旧帖归属
- 论坛宽屏上限放宽至 2560px，主区取消旧 1100px 限制；1920 与 2K 屏完整利用横向空间，左栏、帖子、详情、回复、按钮和辅助信息字号整体上调
- 文件选择改为桌面文件管理器逻辑：普通单击替换选择并切换工作区详情，Ctrl 单击追加/取消，Shift 单击连续范围，复选框继续用于明确多选
- 单击选择延迟到双击判定结束后执行，避免双击进入目录或预览文件时先误改已有选择
- 预览窗口导航文案从“上一张 / 下一张”调整为适用于媒体和文档的“上一个 / 下一个”
- 主页面、音频提取和视频截帧页面统一字体、按钮高度、颜色、边框、对齐与日间/夜间状态
- 工作区卡片重新整理信息层级，文件详情将上传与修改、大小与状态分别合并为同行展示
- 主页面小论坛只显示标题摘要，反应按数量排序；完整正文、楼层和管理操作移入独立论坛
- 发布帖子、发送回复或实时刷新后保留已加载楼层数量，不再重新折叠为前五楼
- “协作设备”更名为“在线设备”，“音频处理”统一更名为“音频提取”
- 逐次上传备份和每日快照均改为默认关闭；仅设置 `DAILY_BACKUP_ENABLED=1` 时启用每日快照

### 修复

- 修复独立论坛左侧“文件管理 / 论坛”导航受吸顶页头的模糊滤镜包含块影响，错误定位到页头左上方并越界的问题；导航移出页头层级，窄屏改为独立横向栏
- 文件管理与论坛页面在离开、浏览器缓存恢复时会显式关闭并重建论坛事件流；服务端同时清理中止、关闭、错误和已失效的连接，避免反复切换后残留连接与心跳造成卡顿
- 修复 MP4/MOV 共用 ffprobe 解封装器时直接取首个别名，导致 `isom/mp41` 品牌的 `.mp4` 文件被误显示为 MOV 的问题
- 修复状态下拉框和 Emoji 选择器被后续内容遮挡或撑高页面的问题
- 修复局域网地址访问时表头及操作栏溢出、按钮无法点击的问题
- 修复按钮未选中却持续高亮、反应按钮重叠、多选控件错位等界面问题
- 对话框遮罩关闭会同时校验鼠标按下位置；从输入框开始框选并拖到窗口外时不再误关闭发帖、预览或移动窗口
- 工作区统计改为服务端实时数据，上传、下载和删除统一按“项”显示，并记录实际数据量
- 修复代码更新、服务重启或切换 8080 / 8082 后需要重新登录的问题

## v3.3.1 (2026-07-22)

### 安全

- NSFW/海外剧改为服务端会话认证；未登录的列表、预览、下载、缩略图和写操作统一返回 401
- 登录会话使用 HttpOnly、SameSite=Strict Cookie，退出和修改密码会立即使旧会话失效
- 聊天、NSFW 登录/退出、密码修改和旧上传入口补齐 CSRF 与 Origin/Referer 校验
- “仅本机”操作改用真实连接地址判断，不再信任客户端可伪造的转发 IP

### 修复

- 旧 NSFW 上传入口复用主上传实现，修复超限仍返回成功、残留截断文件、未等待写入完成和绝对路径前缀越界问题
- 每日快照清单增加源文件 `ctimeMs`，复制与硬链接后都会再次检查源文件状态；旧清单首次升级时会重新复制
- 修复不同访问地址各自保存的旧列宽导致表格最小宽度失控、操作列溢出边界的问题
- 测试版标记移至顶部中央，不再遮挡左上角密码输入
- 静态资源响应增加 `Cache-Control: no-cache`，避免局域网浏览器长期使用旧 CSS/JS

### 工程

- `npm run sync` 在镜像完成后自动提交并推送 `github/` 仓库；预览模式不执行 Git 操作，推送失败会明确返回错误
