# LanShare 交接文档 v3.5.0

## 当前状态

项目正常运行。v3.5.0 已在 dev 增加独立登录页、账号与三级角色系统，并新增独立论坛；主页面小论坛调整为完整论坛的简版预览。自动备份默认关闭。

## 本次完成

- 表格元数据、操作按钮、状态按钮统一使用 `Noto Serif SC` 墨韵主字体
- 列表和宫格操作按钮统一为 30px 高度，文字水平/垂直居中
- 状态按钮箭头参与 Flex 布局并禁止压缩，不再与文字重叠
- 逐次上传备份和每日快照均默认关闭，正常启动不会复制共享文件
- 旧的每日快照实现保留为显式可选功能，仅设置 `DAILY_BACKUP_ENABLED=1` 时启用
- 恢复请求体、单文件上传大小限制并修复旧 NSFW 上传分支
- NSFW 列表、预览、下载、缩略图和写操作必须通过服务端会话认证
- 旧 NSFW 上传统一复用主上传逻辑，超限返回 413 且不会留下截断文件
- 聊天、登录/退出、密码修改和所有文件写接口统一执行 CSRF/Origin 校验
- 快照使用大小、mtime、ctime 和文件标识复核源文件稳定性，硬链接后同样复核
- 自动清洗不同访问地址保存的异常列宽，测试标记移至顶部中央
- 宽屏工作区展示项目总数、进行中的项目、选中文件详情、创建者、当日上传/下载/删除项数与数据量
- 文件列表与宫格支持 Shift 连续范围多选，范围依据当前排序、搜索和状态筛选后的可见顺序，复选框同样适用
- 文件选择采用统一桌面逻辑：普通单击替换为当前项并刷新工作区详情，Ctrl 单击追加/取消单项，Shift 单击替换为锚点范围，Ctrl+Shift 追加范围；复选框保持显式追加多选
- 普通单击等待 120ms 双击判定，双击进入目录或打开预览时会取消待执行的单击选择，避免破坏已有多选
- 工作区单选可预览文件时按需读取媒体信息：图片显示分辨率、像素、色彩和位深；视频/音频显示帧率、码率、时长、封装、编码、采样率和声道
- 媒体信息由 Sharp / 系统 ffprobe 读取，并按路径、大小和修改时间缓存；不会在目录加载时批量探测
- MP4/MOV 容器显示会结合 `major_brand` 与文件扩展名判断，不直接使用 ffprobe 共用解封装器返回的首个别名
- DOCX 作为 `document` 预览类型接入现有预览弹窗；服务端使用 Mammoth 转换正文、列表、表格和内嵌图片，并使用 sanitize-html 白名单清理
- DOCX 页面同时受无脚本 iframe、CSP sandbox、同源权限与 no-store 控制；游客可以查看转换页面，但通用 `/file`、`/download` 和原文件 `/preview` 仍返回 403
- DOCX 默认限制源文件 64MB、转换结果 24MB；结果按路径、大小和修改时间缓存，默认最多 32 项且总量不超过 128MB
- DOCX 预览以内容可读性为目标，分页、文本框及复杂浮动版式可能与 Word 原始排版不同
- 最右侧使用 68px 在线状态条，固定在五个悬浮按钮上方，以单行“头像、账号 ID、绿点”展示；人数较多时内部滚动，工作区和小论坛保持原宽度
- 增加 `/forum` 独立论坛和 `/forum/post/<编号>` 详情路由；主页面左侧导航与小论坛标题右侧均可进入
- 主页面小论坛只读取按最新回复排序的 5 条标题摘要，不再承载正文、楼层和管理操作
- 独立论坛支持热度 / 最新回复 / 最新发布排序、全文搜索、我的帖子、我参与的、我的收藏、未读讨论和分页加载
- 新帖可附带 2–8 个选项的单选投票，支持不限时、1 / 3 / 7 天和自定义 1 小时至 365 天截止；游客可投票，每个账号一票且首次投票后最多改投一次
- 投票结果在帖子详情内显示票数、比例、当前选择和截止状态；帖子列表及主页面小论坛显示投票标识
- 左侧“逛逛分区”增加“投票”筛选标签，筛选所有原分类中的投票帖，不改变帖子的原分区归属
- 论坛分类、投票时长和自定义单位下拉框统一使用墨韵 UI 的自绘箭头与右侧安全间距
- 论坛前台采用“左侧筛选与分类 + 主帖子区”两栏，右侧讨论概要保持移除；窄屏时左栏折为横向可滑动入口
- 分类显示为“茶水间 / 一起开工 / 敲黑板 / 求个支援”，稳定 slug 与旧帖归属不变；发帖可选择“发到”哪个分类
- 收藏按账号保存在 `forum.db`；新帖草稿按账号、回复草稿按账号和帖子保存在当前浏览器，成功发布后清除
- 论坛主布局在 1920 与 2560 宽度内完整展开，主区不再限制为 1100px；左栏、帖子、详情、回复、按钮和辅助文字字号已统一上调
- 发帖支持匿名选项；普通响应不返回匿名作者账号标识，匿名发帖者的后续回复显示为“匿名楼主”
- 完整论坛支持帖子编辑/删除、管理员置顶/锁定、定向楼层回复、楼层编辑/软删除及帖子/楼层 Emoji 反应
- 帖子未读状态按账号保存已读楼层位置，详情打开后清除；浏览量按账号去重
- 论坛变化通过服务端事件流实时推送，主页面另以 30 秒轮询作为断线兜底
- 文件管理与论坛离开页面时主动关闭各自的论坛事件流，浏览器缓存恢复时再重连；服务端会从事件客户端集合中清理中止、关闭、错误和已失效的响应，避免反复切换累积连接
- `forum.db` 启动时自动迁移到结构版本 5，包含分类、收藏、匿名、管理状态、浏览、统计、已读、楼层反应、投票选项、账号投票记录及改投次数
- 独立论坛视觉已与主页面统一：使用相同墨韵日/夜色板、纸纹、衬线字体、左侧竖排导航、直角分隔、透明按钮及无外框 Emoji 反应
- 独立论坛主导航已移出带模糊滤镜的吸顶页头，宽屏保持视口左侧竖排、窄屏切换为页头下方横向栏，不再相对页头定位越界
- 发帖、预览和移动对话框只在鼠标按下与点击都发生于外部遮罩时关闭；输入框内开始框选后拖出窗口不会误关
- 状态下拉框与 Emoji 面板改为上层浮层，不再被后续内容遮挡或撑高布局
- 主页面、音频提取和视频截帧页面统一字体、按钮状态、对齐、颜色及日间/夜间适配
- 独立登录/注册页支持账号、密码、姓名与“记住我”；密码通过 scrypt 加盐保存
- 系统没有管理员时，只有从 `127.0.0.1` / `::1` 注册的首个账号会成为初始管理员
- 初始管理员在数据库中带受保护标记，服务端与管理界面均禁止修改其管理员权限；旧数据库启动时自动锁定最早创建的管理员
- 管理员可重置其他非初始管理员账号的密码，目标账号的全部现有会话会立即失效；本人仍需验证当前密码，初始管理员不接受其他管理员重置
- 管理员可修改其他账号的真实姓名；论坛帖子、楼层、当前聊天和带账号关联的文件创建者会同步显示新姓名，本人姓名仍从账户中心修改
- 在线状态条基于登录会话心跳，同一账号的多个会话合并为一项；详细姓名、角色与设备数放在悬停提示中，停止访问约 45 秒后自动离线
- 普通会话使用 12 小时持久 Cookie，“记住我”为 30 天；服务或浏览器重启后会从 `auth.db` 恢复有效会话
- 正式版 8080 与测试版 8082 使用独立 Cookie 名称，避免同一浏览器在两端口间切换时互相覆盖
- 账号管理支持二次确认后永久删除非初始管理员账号，关联登录会话和该账号的表情反应同时清除，历史帖子与楼层保留
- 注册页的邀请码为选填：留空时新账号为游客，填写当前有效邀请码时直接成为可编辑账号
- 管理员可在账户中心设置一个 6–32 字符的长期邀请码；该码可重复使用，更换或停用前持续有效，更换后旧码立即失效
- 管理员可在账户中心实时分配游客 / 可编辑 / 管理员角色
- 文件、论坛、聊天、日志、回收站和管理 API 均按服务端账号角色校验，不再以浏览器昵称或访问地址代替身份
- 游客可通过独立 `/preview` 流在站内查看图片、音频和视频，并通过 `/api/document-preview` 查看转换后的 DOCX；`/download`、`/file`、批量下载及 DOCX 原文件 `/preview` 仍返回 403，水印仍为后续项
- 帖子作者、回复作者、聊天显示名和后续文件操作记录统一使用登录账号的姓名

## 备份说明

- 当前默认不执行任何自动备份，论坛数据库与账号数据库都不在备份范围内
- 已有备份文件不会自动删除，需要人工确认后再清理
- 如确需临时启用每日快照，设置 `DAILY_BACKUP_ENABLED=1`
- 每日快照目录：`backup/daily/<快照时间>/`
- 服务必须在计划时间保持运行；可用 `DAILY_BACKUP_RUN_ON_START=1` 手动补备份
- 完整项目迁盘时直接迁移整个 `share_web`，默认相对路径会自动跟随新位置
- 账号数据位于运行目录的 `auth.db*`，论坛数据位于 `forum.db*`；保留运行数据时必须停服后一起迁移，代码同步不会复制它们

## 版本管理
- 启动、重启和端口检查统一使用 PowerShell
- dev 端口：8082（标题和版本标记含“测试版”）
- 正式版端口：8080（不含“测试版”标记）
- 同步需经用户确认
- 用户确认同步后，`npm run sync` 会自动提交并推送 `github/` 镜像仓库；推送失败会返回错误
- `.gitignore` 排除 `auth.db*`、`forum.db*`、密码文件和缓存，避免运行数据进入 GitHub
- 同步命令见 CLAUDE.md
