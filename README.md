# 局域网文件服务器 v3.5.0

基于 Node.js 的局域网文件共享与管理工具，采用墨绿色调、衬线字体编辑风 UI。

- 浏览共享目录、上传文件、下载文件、新建文件夹/项目；普通单击切换工作区详情，Ctrl 单击追加选择，Shift 按当前排序连续多选
- DOCX 站内预览：服务端转换正文、表格和内嵌图片，在现有预览窗口中以隔离文档页显示
- 项目状态管理：待开始 / 进行中 / 已完成，`.status` 文件持久化
- 音频提取：从音频或视频导入声音，支持裁剪、变速、倒放、合并，导出 WAV/MP3/FLAC/WebM
- 视频截帧：批量导入视频，帧标记、时间轴定位、批量导出截图
- 墨韵主题：护眼绿底色 + 墨色文字 + Noto Serif 衬线字体，日间/夜间一键切换
- 鼠标拖尾特效：墨印 / 墨痕 / 墨澜 / 墨篆 / 墨染，可自定义颜色
- 自定义右键菜单、文件复制粘贴、背景浮动文字拖拽
- 宽屏工作区：项目总数、进行中项目、选中文件详情、创建者、当日上传/下载/删除项数与数据量
- 媒体详情：单选图片、视频或音频时按需显示分辨率、帧率、码率、时长、编码、采样率、声道及色彩信息
- 最右侧在线状态条：固定在五个悬浮按钮上方，仅显示头像、小号账号 ID 和在线绿点；人数较多时内部滚动，不改变工作区和小论坛宽度
- 小论坛：帖子、定向楼层回复、Emoji 反应、未读提醒、实时刷新及按权限批量删除
- 账号系统：账号、密码、姓名，支持游客 / 可编辑 / 管理员三级角色和“记住我”
- 自动备份默认关闭，避免在项目所在磁盘重复占用共享文件空间
- 海外剧目录使用服务端会话认证，未验证请求不能读取、下载或写入

## 启动

启动、重启和端口检查统一在 PowerShell 中执行。

```powershell
npm start
```

默认监听：

- `http://127.0.0.1:8080`
- `http://你的局域网IP:8080`

默认共享目录：当前目录下的 `shared`

## 账号与权限

- 进入主页面前必须先登录；任何人都可以注册，账号 3–10 个字符、密码 6–16 个字符、姓名 1–20 个字符；邀请码为选填
- 系统尚无管理员时，从 `127.0.0.1` 或本机 `::1` 注册的首个账号自动成为初始管理员
- 初始管理员是受保护账号，其管理员角色不可降级或停用；即使已有其他管理员也不能修改
- 管理员可以重置其他非初始管理员账号的密码，重置后该账号的全部登录会话立即失效；自己的密码仍需在账户中心验证当前密码后修改
- 管理员可以修改其他账号的真实姓名；新姓名会同步到账号、论坛、聊天和已关联的文件创建者显示，自己的姓名仍在账户中心修改
- 管理员可以删除其他普通账号或管理员账号；删除会清除该账号的全部登录会话，但初始管理员不可删除
- 不填写邀请码的新账号从游客开始；填写当前有效邀请码则直接成为可编辑账号
- 管理员可在右上角账户中心的“账号与权限”中设置一个 6–32 字符的邀请码；邀请码可重复使用，更换或停用前持续有效，更换后旧码立即失效
- 宽屏最右侧的 68px 在线状态条按“头像、账号 ID、绿点”单行显示；详细信息使用悬停提示，停止访问约 45 秒后自动离线
- 游客可浏览目录并在站内预览图片、音频、视频和 DOCX，但下载接口及文件修改接口不可用
- 可编辑账号可上传、下载、修改文件及参与论坛；管理员额外拥有账号分配、日志和永久删除等管理权限
- 勾选“记住我”后使用 30 天 HttpOnly 会话 Cookie；未勾选时有效期为 12 小时，服务更新、重启或浏览器重开后仍可恢复有效会话
- 账号密码使用 scrypt 加盐保存于项目目录的 `auth.db`，不会写入浏览器存储或日志
- 游客预览使用独立的站内预览流并隐藏下载、右键和远程播放入口；水印仍为后续项。浏览器播放媒体必然接收媒体数据，因此该限制用于阻止常规下载操作，不等同于 DRM

## 可选环境变量

```powershell
$env:PORT=9000
$env:SHARED_DIR='D:\\共享文件'
$env:NSFW_DIR='D:\\海外剧文件'
$env:BACKUP_DIR='D:\\文件备份'
$env:RECYCLE_DIR='D:\\文件回收站'
$env:AUTH_DB_FILE='D:\\运行数据\\auth.db'
$env:NSFW_SESSION_TTL_MS=43200000  # 海外剧登录有效期，默认 12 小时
$env:MEDIA_INFO_TIMEOUT_MS=15000    # ffprobe 单次探测超时，默认 15 秒
$env:MEDIA_INFO_CACHE_LIMIT=500     # 服务端媒体信息内存缓存项数
$env:DOCUMENT_PREVIEW_MAX_SIZE=67108864        # DOCX 源文件预览上限，默认 64MB
$env:DOCUMENT_PREVIEW_MAX_HTML_SIZE=25165824   # DOCX 转换结果上限，默认 24MB
$env:DOCUMENT_PREVIEW_CACHE_LIMIT=32            # DOCX 内存缓存项数
$env:DOCUMENT_PREVIEW_CACHE_MAX_BYTES=134217728 # DOCX 内存缓存总量，默认 128MB
npm start
```

自动备份默认关闭。只有明确需要时才可临时启用，并调整执行时间：

```powershell
$env:DAILY_BACKUP_ENABLED=1
$env:DAILY_BACKUP_HOUR=23
$env:DAILY_BACKUP_MINUTE=55
$env:DAILY_BACKUP_RUN_ON_START=1  # 启动后立即执行一次，适合手动补备份
npm start
```

## 可选备份（默认关闭）

- 正常启动不会创建每日快照，也不会备份小论坛数据库或账号数据库
- 默认备份位置：`backup/daily/<快照时间>/`
- 同时备份 `shared` 和 `shared_NSFW`
- 首次快照完整复制；后续快照只复制新增或修改文件
- 未变化文件在备份快照之间创建 NTFS 硬链接，不重复占用物理空间
- 复制或创建硬链接期间检测到大小、修改时间或状态时间变化的文件会跳过，并记录到 `.snapshot.json`
- 默认不删除现有旧备份，也不自动清理历史快照
- 如确需启用，设置 `$env:DAILY_BACKUP_ENABLED=1`；设置 `$env:PER_UPLOAD_BACKUP_ENABLED=1` 可临时恢复旧的逐次上传备份

## 整体迁移到新硬盘

项目的共享目录、备份、回收站、日志和静态资源默认都基于项目目录定位。迁移整个项目时，建议先复制、停服、最终同步并验证，再从新位置启动；通常不需要修改代码。

- 新硬盘建议使用 NTFS
- 停止 8080 和 8082 后再做最终同步，避免遗漏迁移期间的上传或删除
- `thumb_cache` 包含旧绝对路径，可不迁移，由服务自动重新生成
- 需要保留账号和论坛内容时，应在停服后随项目迁移 `auth.db*` 与 `forum.db*`；代码同步不会复制运行时数据库
- 如设置过 `SHARED_DIR`、`NSFW_DIR`、`BACKUP_DIR` 或 `RECYCLE_DIR`，迁移后需要清除或更新这些绝对路径
- 若手动启用并生成过硬链接快照，不要用普通复制方式随意搬运备份历史，以免硬链接被展开成多份文件

## 说明

- 上传通过网页端读取文件后发送到本地服务，适合局域网内部使用
- 列表与宫格使用统一选择逻辑：普通单击仅选择当前项并切换工作区详情；Ctrl 单击或复选框追加/取消单项；Shift 单击按当前可见排序和筛选结果选择连续范围
- 单击操作延迟到双击判定结束后执行，双击打开目录或预览文件时不会先误改多选状态
- 媒体详情只在单选可预览文件时按需探测，并按文件路径、大小和修改时间缓存，不会在打开目录时批量扫描
- 图片信息使用 Sharp 读取；视频和音频信息使用系统 `ffprobe`，因此需要 FFmpeg 工具可用
- DOCX 通过 Mammoth 提取语义化内容，再经 HTML 白名单清理并放入无脚本 iframe；正文、列表、表格和内嵌图片可预览，分页、文本框和复杂浮动版式不保证与 Word 逐像素一致
- DOCX 转换结果按文件路径、大小和修改时间缓存；游客只能读取转换后的隔离页面，`/file`、`/download` 和原文件 `/preview` 仍不可访问
- 服务只允许文件操作落在 `shared` 或已认证的 `shared_NSFW` 内，并对主、旧版上传入口统一执行路径越界检查
- 海外剧登录使用 HttpOnly、SameSite=Strict 会话 Cookie；退出或修改密码后旧会话立即失效
- 主账号会话同样使用 HttpOnly、SameSite=Strict Cookie；8080 与 8082 使用独立 Cookie，角色调整对现有会话立即生效
- 局域网其他设备无法访问通常是 Windows 防火墙未放行对应端口
