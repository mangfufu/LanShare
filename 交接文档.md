# LanShare 交接文档 v3.5.0

## 当前状态

项目正常运行。v3.5.0 完成宽屏工作区、小论坛、实时工作数据和整套 UI 一致性升级；自动备份现已默认关闭，避免在项目盘重复占用大量空间。

## 本次完成

- 表格元数据、操作按钮、状态按钮统一使用 `Noto Serif SC` 墨韵主字体
- 列表和宫格操作按钮统一为 30px 高度，文字水平/垂直居中
- 状态按钮箭头参与 Flex 布局并禁止压缩，不再与文字重叠
- 逐次上传备份和每日快照均默认关闭，正常启动不会复制共享文件
- 旧的每日快照实现保留为显式可选功能，仅设置 `DAILY_BACKUP_ENABLED=1` 时启用
- 恢复请求体、单文件上传大小限制并修复旧 NSFW 上传分支
- NSFW 列表、预览、下载、缩略图和写操作必须通过服务端会话认证
- 旧 NSFW 上传统一复用主上传逻辑，超限返回 413 且不会留下截断文件
- 聊天、登录/退出、密码修改和所有文件写接口统一执行 CSRF/Origin 校验
- 快照使用大小、mtime、ctime 和文件标识复核源文件稳定性，硬链接后同样复核
- 自动清洗不同访问地址保存的异常列宽，测试标记移至顶部中央
- 宽屏工作区展示项目总数、进行中的项目、选中文件详情、创建者、当日上传/下载/删除项数、数据量与在线设备
- 小论坛支持帖子、定向楼层回复、每次加载五楼、保持已加载楼层、Emoji 反应、实时刷新和当前浏览器未读红点
- 贴主可删除自己的帖子，本机管理员可删除全部帖子；支持多选、全选和批量删除
- 状态下拉框与 Emoji 面板改为上层浮层，不再被后续内容遮挡或撑高布局
- 主页面、音频提取和视频截帧页面统一字体、按钮状态、对齐、颜色及日间/夜间适配

## 备份说明

- 当前默认不执行任何自动备份，小论坛数据库也不在备份范围内
- 已有备份文件不会自动删除，需要人工确认后再清理
- 如确需临时启用每日快照，设置 `DAILY_BACKUP_ENABLED=1`
- 每日快照目录：`backup/daily/<快照时间>/`
- 服务必须在计划时间保持运行；可用 `DAILY_BACKUP_RUN_ON_START=1` 手动补备份
- 完整项目迁盘时直接迁移整个 `share_web`，默认相对路径会自动跟随新位置

## 版本管理
- 启动、重启和端口检查统一使用 PowerShell
- dev 端口：8082（标题和版本标记含“测试版”）
- 正式版端口：8080（不含“测试版”标记）
- 同步需经用户确认
- 用户确认同步后，`npm run sync` 会自动提交并推送 `github/` 镜像仓库；推送失败会返回错误
- 同步命令见 CLAUDE.md
